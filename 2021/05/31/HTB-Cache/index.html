枚举

web枚举

网站存在首页

作者栏发现用户ASH，并且发现了主机名，添加到/etc/hosts里

内容提示了ASH是一个安全研究员，它创建了HMS系统，这是一个提示

发现登陆页面，尝试常见弱口令

发现BurpSuite里没有请求包，所以这有可能是Javascript处理的

查看源代码，发现view-source:http://10.10.10.188/jquery/functionality.js

发现了硬编码凭据ash/H@v3_fun

输入凭据后页面跳转，但是没有更多的信息

对虚拟主机名进行枚举。这里很坑，需要把htb添加到hosts里

然后对htb进行虚拟主机名枚举

得到hms.htb，添加到hosts里

通过hms.htb访问，发现openEMR，下方提示2018，看起来版本较老

渗透

谷歌搜索发现了关于OpenEMR的漏洞手册，里面记录了大量漏洞:链接

首先让我感兴趣的是版本枚举

未授权用户可以通过访问admin.php得到版本号，尝试访问http://hms.htb/admin.php

或者sql_patch.php似乎也可以

访问http://hms.htb/sql_patch.php得到

有了版本就可以过滤掉大量无关的公开漏洞

不幸的是多数需要身份认证

身份绕过

从漏洞报告PDF中，发现了Patient Portal Authentication Bypass，可以使我们绕过身份验证

利用过程很简单，首先访问首页，302跳转后会给我们设置一个cookie

之后访问http://hms.htb/portal/account/register.php，这里由于逻辑问题，会给我一个有效的session会话

然后我们就相当于通过了认证，刷新页面，发现确实多了一个session

sql注入

之后就可以顺着PDF文档阅读，发现了大量SQL注入漏洞(需要身份验证)，我们可以对它们发起攻击，比如

如果没有经过身份认证绕过，该漏洞无法利用

绕过后的报错注入生效

直接使用sqlmap就行

这里使用Burpsuite的Intruder快速完成报错注入

前面都是正常操作

Start attack，攻击完成后，为了方便查看，添加Grep - Extract

之后可以直接在Intruder页面看到结果

反正随便怎么操作就能获取加密的哈希了

hashid猜测出密码可能为bcrypt

使用john破解bcrypt Blowfish

得到xxxxxx

使用凭据openemr_admin/xxxxxx可以登录网站

shell

有了管理员凭据，可以在后台花式上传shell。为了快速获得RCE，我使用之前EDB里找到的45161.py
